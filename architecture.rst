================================
AI Architect: Application Architecture
================================

Overview
========

This document provides a detailed blueprint of the AI Architect application's file and class structure. It is intended to be a technical reference for developers and AI models to ensure that all code implementation aligns with the planned architecture. The core idea is that AI agents interact directly with a project's file system, rather than a central configuration file.

Directory Structure
===================

Application source code project directory structure

.. code-block:: bash

    /
    ├── src/
    │   └── ai_architect/
    │       ├── __init__.py
    │       ├── data_models/
    │       │   ├── __init__.py
    │       │   └── plan.py
    │       ├── services/
    │       │   ├── __init__.py
    │       │   ├── agent_runner.py
    │       │   ├── cloudflare_service.py
    │       │   ├── deployment_service.py
    │       │   ├── file_system_service.py
    │       │   ├── git_service.py
    │       │   └── language_model_service.py
    │       ├── executor.py
    │       ├── orchestrator.py
    │       └── planner.py
    └── main.py

Website project directory

.. code-block:: bash

    /my-website-project/
    ├── .dev/                         # Internal state files for the AI Architect CLI
    │   ├── research/                 # Agent research output (.md files)
    │   └── history/                  # Stores past task.yml plans
    ├── agents/                       # Agent profiles (.yml files)
    │   ├── content.agent.yml
    │   ├── designer.agent.yml
    │   └── capabilities.yml          # "Menu" of available agents for the Planner
    ├── build/                        # Output directory for the compiled static site
    ├── content/                      # Content files (e.g., YAML content)
    ├── src/                          # Astro JS source (e.g., HTML templates and components)
    └── public/                       # Publicly accessible assets (images, CSS, JS, etc.)


Core Components
===============

main.py
-------

**Purpose**: The main entry point for the CLI application. It parses command-line arguments and initializes the ``Orchestrator``.

src/ai_architect/orchestrator.py
--------------------------------

**Purpose**: Routes user commands to the appropriate workflow (AI-driven or deterministic). It acts as the main controller of the application.

**Class**: ``Orchestrator``

  * **Description**: Parses CLI arguments and orchestrates the application's workflow by invoking the appropriate agents or services.
  * **Methods**:
    * ``__init__(self, argv: list[str])``: Initializes the orchestrator with command-line arguments.
    * ``run(self) -> None``: The main execution method. It parses arguments and decides which workflow to run.

src/ai_architect/planner.py
---------------------------

**Purpose**: Contains the AI-driven planner that translates fuzzy user prompts into a deterministic plan.

**Class**: ``Planner``

  * **Description**: Uses a language model to generate a ``Plan`` object from a user's natural language prompt.
  * **Properties**:
    * ``lm_service: LanguageModelService``: Service for interacting with the AI model.
  * **Methods**:
    * ``__init__(self, lm_service: LanguageModelService)``: Initializes the planner with a language model service.
    * ``generate_plan(self, prompt: str, project_path: str) -> Plan``: Takes a user prompt and project path, scans the file system for context, and generates a step-by-step plan.

src/ai_architect/executor.py
----------------------------

**Purpose**: Executes a plan generated by the ``Planner``.

**Class**: ``Executor``

  * **Description**: A non-AI workflow engine that iterates through tasks in a ``Plan`` and uses the ``AgentRunner`` service to execute them.
  * **Properties**:
    * ``agent_runner: AgentRunner``: The service responsible for executing agent tasks.
    * ``project_path: str``: The root path of the user's project.
  * **Methods**:
    * ``__init__(self, project_path: str, agent_runner: AgentRunner)``: Initializes the executor.
    * ``execute_plan(self, plan: Plan) -> None``: Loops through the tasks in the plan and calls the ``agent_runner`` to execute each one.

Data Models
===========

src/ai_architect/data_models/plan.py
------------------------------------

**Purpose**: Defines the structure for ``Plan`` and ``Task`` objects.

**Class**: ``Task``

  * **Description**: A data class representing a single step in a plan.
  * **Properties**:
    * ``name: str``: A human-readable name for the task.
    * ``agent: str``: The name of the agent class required to execute the task.
    * ``notice: str``: A message to display to the user before executing the task.
    * ``params: dict``: A dictionary of parameters to be passed to the agent's ``execute`` method.

**Class**: ``Plan``

  * **Description**: A data class that contains a list of ``Task`` objects.
  * **Properties**:
    * ``plan_name: str``: The name of the overall plan.
    * ``tasks: list[Task]``: A list of tasks to be executed in sequence.
  * **Methods**:
    * ``to_yaml(self) -> str``: Serializes the plan to a YAML string.
    * ``from_yaml(cls, yaml_string: str) -> Plan``: A class method to create a ``Plan`` instance from a YAML string.


src/ai_architect/agents/base_agent.py
-------------------------------------

**Purpose**: Defines the abstract base class for all specialist agents.

**Class**: ``BaseAgent(ABC)``

  * **Description**: An abstract base class that all specialist agents must inherit from. It ensures a consistent interface for the ``Executor``.
  * **Methods**:
    * ``@abstractmethod execute(self, task: Task, project: Project) -> Project``: The primary method for an agent. It takes a task and the project state, performs an action, and
returns the (potentially modified) project state.

src/ai_architect/agents/content_manager.py
------------------------------------------

**Purpose**: An agent for creating and modifying the content of ``project.yml``.

**Class**: ``ContentManagerAgent``

  * **Inherits from**: ``BaseAgent``
  * **Description**: Writes, updates, or refactors YAML content in ``project.yml`` based on a prompt.
  * **Properties**:
    * ``lm_service: LanguageModelService``: Service for interacting with the AI model.
    * ``fs_service: FileSystemService``: Service for file I/O.
  * **Methods**:
    * ``__init__(self, lm_service: LanguageModelService, fs_service: FileSystemService)``: Initializes the agent.
    * ``execute(self, task: Task, project: Project) -> Project``: Builds a prompt for the language model, gets the new YAML content, writes it to ``project.yml``, and reloads the
project state.

src/ai_architect/agents/illustrator.py
--------------------------------------

**Purpose**: An agent for generating images.

**Class**: ``IllustratorAgent``

  * **Inherits from**: ``BaseAgent``
  * **Description**: Generates images from text prompts and saves them to the ``/public/images`` directory.
  * **Properties**:
    * ``image_generation_service: Any``: A service for interacting with an image generation API (e.g., DALL-E, Midjourney).
  * **Methods**:
    * ``__init__(self, image_generation_service: Any)``: Initializes the agent.
    * ``execute(self, task: Task, project: Project) -> Project``: Calls the image generation service for each prompt in the task's parameters and saves the resulting images to the
specified file paths.

src/ai_architect/agents/compiler.py
-----------------------------------

**Purpose**: A deterministic agent for compiling the project.

**Class**: ``CompilerAgent``

  * **Inherits from**: ``BaseAgent``
  * **Description**: A hard-coded agent that transforms the ``project.yml`` data into a static website in the ``/src`` directory.
  * **Methods**:
    * ``__init__(self)``: Initializes the agent.
    * ``execute(self, task: Task, project: Project) -> Project``: Reads the project state and generates the necessary HTML, CSS, and JS files in the ``/src`` directory. The project
state is not modified.

src/ai_architect/services/language_model_service.py
---------------------------------------------------
========

src/ai_architect/services/language_model_service.py
---------------------------------------------------

**Purpose**: Provides a wrapper around an external language model API.

**Class**: ``LanguageModelService``

  * **Description**: Abstracts the details of calling a language model (e.g., Gemini, OpenAI API).
  * **Methods**:
    * ``__init__(self, api_key: str)``: Initializes the service with an API key.
    * ``query(self, prompt: str) -> str``: Sends a prompt to the language model and returns the text response.

src/ai_architect/services/file_system_service.py
------------------------------------------------

**Purpose**: A helper for all file system I/O operations.

**Class**: ``FileSystemService``

  * **Description**: Provides a centralized and mockable interface for reading from and writing to the file system.
  * **Methods**:
    * ``read_file(self, file_path: str) -> str``: Reads the content of a file and returns it as a string.
    * ``write_file(self, file_path: str, content: str) -> None``: Writes content to a specified file.

src/ai_architect/services/git_service.py
----------------------------------------

**Purpose**: A wrapper for Git commands.

**Class**: ``GitService``

  * **Description**: Abstracts Git operations like committing and pushing.
  * **Methods**:
    * ``__init__(self, repo_path: str)``: Initializes the service with the path to the Git repository.
    * ``commit_and_push(self, branch: str, message: str) -> None``: Commits all changes and pushes to the specified remote branch.

src/ai_architect/services/cloudflare_service.py
-----------------------------------------------

**Purpose**: A wrapper for the Cloudflare API.

**Class**: ``CloudflareService``

  * **Description**: Interacts with the Cloudflare API, for tasks like checking build statuses.
  * **Methods**:
    * ``__init__(self, api_token: str)``: Initializes the service with a Cloudflare API token.
    * ``wait_for_build(self, project_name: str, environment: str) -> None``: Polls the Cloudflare API until a deployment build is complete.

src/ai_architect/services/deployment_service.py
-----------------------------------------------

**Purpose**: Contains the logic for deterministic deployment commands.

**Class**: ``DeploymentService``

  * **Description**: Handles the hard-coded workflows for ``--publish`` and ``--preview`` commands.
  * **Properties**:
    * ``agent_runner: AgentRunner``: The service for running agent tasks, used here to run the compiler agent.
    * ``git_service: GitService``: Service for Git operations.
    * ``cloudflare_service: CloudflareService``: Service for Cloudflare operations.
  * **Methods**:
    * ``__init__(self, agent_runner: AgentRunner, git_service: GitService, cloudflare_service: CloudflareService)``: Initializes the service with its dependencies.
    * ``run_production_deployment(self) -> None``: Executes the sequence for a production deployment: run compile task, commit, push, and wait for build.
